Core Modules (by Criticality)
CRITICAL
Rust Daemon
Persistent database connections, embedding cache management, connection pooling. 20-50x speedup vs process-per-request model. Single source of truth for repository state.
CRITICAL
Tree-sitter Parser
AST-based code extraction for 10+ languages. Enables accurate function/class/structure boundary detection without regex. Foundation for semantic analysis.
CRITICAL
Search Fusion Pipeline
FTS5 (BM25) + vector similarity (cosine) + Reciprocal Rank Fusion (k=60). Hybrid approach eliminates false negatives. Semantic ranking with kind boost and recency bonus.
CRITICAL
Embedding Cache
SHA-256 content deduplication. 70-90% API call reduction. Multi-dimension support (768/1024/1536). In-memory and persistent caching layers.
HIGH
Worktree Management
Isolated git worktrees per branch, automatic cleanup, config auto-copy. Prevents git state collisions and enables parallel workspace analysis.
HIGH
Agent Orchestration
iTerm2 pane management or headless execution mode. Structured message protocol for tool coordination and state synchronization.
MEDIUM

Unique & Distinguishing Aspects
Content-Aware Deduplication
SHA-256 hashing on code chunks eliminates redundant embeddings across repos and branches. Identical code reuses cached vectors regardless of location or worktree.
Branch-Aware Indexing
Per-worktree isolation prevents branch state collisions. Index updates scoped to specific branches; enables parallel analysis and incremental sync.
Relationship Graph Traversal
chunk_edges graph (imports, calls, defines, tests, routes) enables context-aware result assembly. Returns semantic dependencies alongside search matches.
Multi-Provider Embedding
Provider-agnostic abstraction: Ollama (local/default), OpenAI, Google Vertex AI. Auto-detection and dimension flexibility (768/1024/1536). Cost optimization with local defaults.
Monorepo-Native Support
pnpm/npm/yarn workspace-aware indexing. Multi-workspace chunking and cross-workspace relationship traversal. Scales to large polyglot monorepos.
Hybrid Search Fusion
Reciprocal Rank Fusion combines BM25 (lexical) + cosine similarity (semantic). Eliminates false negatives from vector-only search. Semantic re-ranking with kind boost and recency.
