# Project: SCIP Schema & Import Foundation

## Project Summary

Create the database schema and import pipeline to load a single `.scip` file (from scip-typescript) into SQLite. This establishes the data foundation for code intelligence without any query or MCP complexity. The SCIP (Sourcegraph Code Intelligence Protocol) format contains precise symbol definitions, references, and relationships extracted by language-aware indexers.

Success is a populated SQLite database that can be inspected manually and serves as the foundation for subsequent query and MCP integration projects.

### Edge Cases
- Empty `.scip` file: Create empty tables, log warning
- Malformed protobuf: Fail with clear error message
- Duplicate symbols: Use UPSERT, last occurrence wins
- Very large files (>1GB): Stream processing, don't load all in memory

## Technical Design

### SQLite Schema

```sql
-- Documents (source files)
CREATE TABLE scip_documents (
  id INTEGER PRIMARY KEY,
  relative_path TEXT NOT NULL UNIQUE,
  language TEXT,
  occurrences_count INTEGER DEFAULT 0
);

-- Symbols (unique code entities)
CREATE TABLE scip_symbols (
  id INTEGER PRIMARY KEY,
  symbol TEXT NOT NULL UNIQUE,  -- SCIP symbol string
  kind TEXT,                     -- function, class, method, etc.
  display_name TEXT,
  documentation TEXT,
  signature TEXT
);

-- Occurrences (symbol usages at specific locations)
CREATE TABLE scip_occurrences (
  id INTEGER PRIMARY KEY,
  document_id INTEGER NOT NULL REFERENCES scip_documents(id),
  symbol_id INTEGER NOT NULL REFERENCES scip_symbols(id),
  start_line INTEGER NOT NULL,
  start_col INTEGER NOT NULL,
  end_line INTEGER NOT NULL,
  end_col INTEGER NOT NULL,
  role INTEGER NOT NULL,  -- 1=definition, 2=reference, etc.
  UNIQUE(document_id, symbol_id, start_line, start_col)
);

-- Relationships between symbols
CREATE TABLE scip_relationships (
  id INTEGER PRIMARY KEY,
  from_symbol_id INTEGER NOT NULL REFERENCES scip_symbols(id),
  to_symbol_id INTEGER NOT NULL REFERENCES scip_symbols(id),
  kind TEXT NOT NULL  -- implements, extends, etc.
);

-- Indexes for fast lookups
CREATE INDEX idx_occurrences_document ON scip_occurrences(document_id);
CREATE INDEX idx_occurrences_symbol ON scip_occurrences(symbol_id);
CREATE INDEX idx_occurrences_position ON scip_occurrences(document_id, start_line, start_col);
CREATE INDEX idx_symbols_name ON scip_symbols(symbol);
```

### Import Algorithm

```
1. Parse .scip file using scip crate
2. Begin SQLite transaction
3. For each Document in Index:
   a. Insert into scip_documents
   b. For each Symbol in Document.symbols:
      - UPSERT into scip_symbols
   c. For each Occurrence in Document.occurrences:
      - Resolve symbol_id
      - Insert into scip_occurrences
4. For each external_symbol in Index:
   - UPSERT into scip_symbols (for cross-repo refs)
5. Commit transaction
6. Run ANALYZE for query optimizer
```
