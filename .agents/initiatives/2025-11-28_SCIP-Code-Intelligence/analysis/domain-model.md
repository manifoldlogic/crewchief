# Domain Model: SCIP-Based Code Intelligence

## Core Entities

### Index Artifacts

```
┌─────────────────┐
│   SCIP Index    │ ─── Protobuf file generated by indexer
├─────────────────┤
│ metadata        │     Language, version, project root
│ documents[]     │     Source files in the index
│ external_syms[] │     Symbols from dependencies
└─────────────────┘
         │
         ▼
┌─────────────────┐
│    Document     │ ─── Single source file
├─────────────────┤
│ relative_path   │     Path from project root
│ language        │     File language
│ occurrences[]   │     Symbol usages in this file
│ symbols[]       │     Symbols defined in this file
└─────────────────┘
```

### Symbol Model

```
┌─────────────────┐
│     Symbol      │ ─── Unique code entity identifier
├─────────────────┤
│ symbol_id       │     Globally unique string (SCIP format)
│ display_name    │     Human-readable name
│ kind            │     Function, class, method, etc.
│ signature       │     Type signature
│ documentation   │     Docstring/comments
└─────────────────┘
         │
         ▼
┌─────────────────┐
│   Occurrence    │ ─── Location where symbol appears
├─────────────────┤
│ symbol_id       │     Reference to symbol
│ range           │     Start/end position in file
│ role            │     Definition or Reference
│ syntax_kind     │     Import, call, type annotation, etc.
└─────────────────┘
```

## Boundaries

### What SCIP Captures

- Symbol definitions and their locations
- All references (usages) of each symbol
- Symbol metadata (kind, signature, docs)
- Import/export relationships
- Type hierarchy (implements, extends)

### What SCIP Does NOT Capture

- Runtime behavior (dynamic dispatch targets)
- Conditional compilation paths
- Dead code detection
- Performance characteristics
- Test coverage

### Boundary with Maproom

```
┌─────────────────────────────────────────────────────────────────┐
│                         Maproom Database                         │
├─────────────────────┬───────────────────────────────────────────┤
│   Existing Tables   │              SCIP Tables (new)            │
├─────────────────────┼───────────────────────────────────────────┤
│ chunks              │ scip_symbols                              │
│ chunk_embeddings    │ scip_occurrences                          │
│ chunk_edges         │ scip_documents                            │
│ worktrees           │ scip_indexes (metadata)                   │
│ repositories        │                                           │
└─────────────────────┴───────────────────────────────────────────┘
                              │
                              ▼
                    Shared: worktree_id FK
```

SCIP tables link to existing Maproom schema via `worktree_id`, enabling:
- Joint queries across semantic search and code intelligence
- Unified repository/worktree management
- Consistent storage location

## Interactions

### Index Time Flow

```
1. User runs `maproom scan`
2. Language detection identifies project types
3. Appropriate indexer invoked (scip-typescript, rust-analyzer, etc.)
4. Indexer produces .scip file (protobuf)
5. Import pipeline parses protobuf
6. Data written to scip_* tables
7. Old index data for worktree purged
```

### Query Time Flow

```
1. Agent calls MCP tool (e.g., scip_goto_definition)
2. Tool receives file path + position
3. Query engine finds occurrence at position
4. If occurrence is reference → find definition occurrence
5. Return file + line + column to agent
```

### Cross-Entity Queries

| Query | Tables Involved | Join Path |
|-------|-----------------|-----------|
| Definition of symbol | occurrences → symbols | symbol_id |
| References to symbol | symbols → occurrences | symbol_id, is_definition=false |
| Symbols in file | documents → occurrences → symbols | document_id |
| All files with symbol | symbols → occurrences → documents | symbol_id |
