# .agents/knowledge - Working Knowledge Base

Research, specs, and technical knowledge supporting **active project work**.

## Purpose

This directory stores domain knowledge that agents gather and reference during project planning and execution. It's a working repository - content here may be incomplete, evolving, or project-specific.

## What Belongs Here

- Technical research for ongoing projects
- Product vision documents and specs
- Domain-specific notes (maproom/, cli/, research/)
- Reference material gathered during investigation
- Knowledge that may later be synthesized into `/docs/`

## What Does NOT Belong Here

- Finalized, permanent documentation (goes in `/docs/`)
- Active project plans and tickets (go in `.agents/projects/`)
- Agent definitions (go in `.agents/agents/`)
- Process templates (go in `.agents/reference/`)

## Lifecycle

When a project completes, relevant knowledge should be:
1. Synthesized into permanent documentation in `/docs/`
2. Kept here only if it supports future related work
3. Deleted if obsolete or superseded
