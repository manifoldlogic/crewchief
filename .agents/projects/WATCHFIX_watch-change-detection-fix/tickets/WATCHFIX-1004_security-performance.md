## Summary
Add file size limits to prevent DoS attacks and optional symlink detection for security awareness. These safeguards protect the watch command from resource exhaustion and provide visibility into potential security concerns.

## Background
Per security review (`.agents/projects/WATCHFIX_watch-change-detection-fix/planning/security-review.md`), the watch command is vulnerable to DoS attacks via very large files. For example, a 1GB file triggers OOM when `fs::read_to_string()` loads the entire file into memory. Additionally, symlink following can index files outside the repository without user awareness.

This ticket implements Phase 4 of the WATCHFIX project plan (security & performance improvements) by adding pragmatic safeguards without breaking legitimate use cases. The 10MB limit protects against resource exhaustion while accommodating typical source code files.

## Acceptance Criteria
- [ ] Files larger than 10MB are skipped with warning log message
- [ ] File size check happens before reading file content
- [ ] Symlinks are detected and logged with warning message
- [ ] Symlinks are allowed (not rejected) - just logged for awareness
- [ ] No panics or OOM crashes on large files
- [ ] Performance unchanged for normal-sized files (< 10MB)
- [ ] Configuration constant `MAX_FILE_SIZE_BYTES` easy to adjust

## Technical Requirements

### File Size Check
- **Location**: `crates/maproom/src/incremental/processor.rs`
- **Method**: `index_new_file()` - add check before `fs::read_to_string()`
- **Method**: `update_file()` - add same check
- **Constant**: Add `const MAX_FILE_SIZE_BYTES: u64 = 10 * 1024 * 1024; // 10MB` at file level

### Check Before Reading
```rust
// In index_new_file(), before reading file content:
let metadata = fs::metadata(path)
    .with_context(|| format!("Failed to get file metadata: {}", path.display()))?;

if metadata.len() > MAX_FILE_SIZE_BYTES {
    warn!(
        path = %path.display(),
        size_mb = metadata.len() / (1024 * 1024),
        "File too large to index, skipping"
    );
    return Ok(()); // Skip gracefully, don't error
}
```

### Symlink Detection (Optional)
```rust
// After size check:
if metadata.file_type().is_symlink() {
    warn!(
        path = %path.display(),
        "Indexing symlink - resolved path may be outside repository"
    );
    // Continue processing - just log awareness
}
```

### Error Handling
- Return `Ok(())` not `Err()` for oversized files (graceful skip)
- Log at `warn!` level (visible without `RUST_LOG=debug`)
- No panics or crashes on edge cases

## Implementation Notes

### Technical Approach
1. **Check metadata BEFORE reading**: This prevents OOM by inspecting file size before loading into memory
2. **Graceful skip for large files**: Return `Ok(())` to continue processing other files, not `Err()` which might stop the entire watch
3. **Warning-level logging**: Users should see these messages without verbose debug logging
4. **Cross-platform compatibility**: Both `fs::metadata()` and `file_type().is_symlink()` are cross-platform

### Why 10MB?
- Source code files are typically < 1MB
- 10MB accommodates large autogenerated files, package-lock.json, etc.
- Protects against accidental indexing of binary artifacts (images, videos, compiled binaries)
- Easy to adjust via constant if needed

### Symlink Behavior
- **Log but allow**: Users should be aware symlinks might point outside repo, but blocking them breaks legitimate use cases (node_modules, vendor dirs)
- **Future consideration**: Could add `--follow-symlinks` flag for explicit control

### Files Affected
Apply same checks to both methods:
- `index_new_file()` - when watching detects new file
- `update_file()` - when watching detects modified file

### Performance Impact
- `fs::metadata()` is fast (no file content read)
- Adds ~1-2ms per file (negligible)
- For files < 10MB: no behavior change

## Dependencies
None - this is an independent security improvement

## Risk Assessment

- **Risk**: 10MB limit might be too restrictive for some repositories
  - **Mitigation**: Easy to adjust `MAX_FILE_SIZE_BYTES` constant; document in code comments

- **Risk**: Symlink detection behavior might not work on all platforms
  - **Mitigation**: `file_type().is_symlink()` is standard library cross-platform API; test on Linux/macOS/Windows if possible

- **Risk**: Skipping large files silently might confuse users
  - **Mitigation**: Log at `warn!` level with file path and size; visible in normal operation
