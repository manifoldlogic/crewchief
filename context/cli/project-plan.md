# CrewChief Implementation Plan

This plan operationalizes `context/specification.md` and `context/technology.md` into executable work. It is organized into phases with milestones, detailed tasks, interfaces, directory layout, and acceptance criteria. Package manager: pnpm.

### High-Level Milestones

- M0: Repo scaffolding and dev ergonomics ready
- M1: Core foundation (worktrees, tmux session control, config)
- M2: Agent integration (Claude/Gemini runners, message bus, result capture)
- M3: Orchestration (task distribution, evaluation, merge strategies, competition basics)
- M4: Polish (setup wizard, advanced competition, performance, recovery)

## Phase-by-Phase Execution

### M0: Scaffolding (Day 0â€“0.5)

- Create `package.json`, `tsconfig.json`, pnpm scripts:
- `dev`: tsx watch CLI
- `build`: tsup build to `dist`
- `lint`, `format`, `test`
- Bin shim `bin/crewchief` with shebang -> `dist/cli.js`
- Minimal `src/cli/index.ts` with `--version` and `--help`
- Logger and exec utilities
- CI skeleton (optional, GitHub Actions)

Acceptance: `pnpm dev` runs CLI help; `pnpm build` produces `dist`; no type/lint errors.

### M1: Core Foundation (Week 1)

- Git worktree wrappers (`src/git/worktrees.ts`)
  - `initRepository()` â€” ensure clean state, create `.crewchief/worktrees`
  - `createWorktree(name, baseBranch)` â€” using `simple-git`
  - `listWorktrees()` â€” parse `git worktree list --porcelain`
  - `cleanWorktrees(predicate)` â€” remove non-active worktrees
  - Persist minimal `WorktreeConfig` under `.crewchief/meta/<name>.json`

- Tmux session management (`src/tmux/tmux.service.ts`)
  - Detect tmux binary, create session, split panes
  - API: `createPane(layout)`, `sendKeys`, `captureOutput`, `closePane`, `resizePane`
  - Layout helpers for orchestrator + agents

- Configuration (`src/config/*`)
  - Zod schema for `crewchief.config.ts` per spec
  - Loader that resolves TS config via dynamic import; apply defaults

- CLI commands
- `init`, `worktree create/list/clean`, `session start`

Acceptance:

- Create/list/clean worktrees successfully on a sample repo
- Start a tmux session with orchestrator pane and one agent pane
- Config is validated and accessible across commands

### M2: Agent Integration (Week 2)

- Agent discovery & registry
- Scan `.claude/agents/`, `.gemini/agents/` for definitions
- Registry entries for common roles (project-manager, backend, frontend)

- Agent runner (`src/agents/runner.ts`)
- Spawn external CLIs via `node-pty` into dedicated tmux panes
- Environment preparation (API keys from env, config overrides)
- Result capture: stdout JSONL frames with message envelopes

- Message bus (`src/bus/message.bus.ts`)
- In-memory event emitter; `send`, `onMessage`, `waitForResponse`
- JSONL framing utilities, backpressure safeguards, timeouts

- CLI commands
- `agent spawn <type> <task>` creates worktree, pane, and starts runner
- `agent message <agent-id> <message>` routes to runner stdin
- `agent close <agent-id>` tears down pane, optional merge flag (deferred)

Acceptance:

- Spawn Claude/Gemini placeholder agents that echo messages and write JSONL
- Message round-trips: orchestrator -> agent -> orchestrator
- Captured outputs persisted under `.crewchief/runs/<agent-id>/`

### M3: Orchestration (Week 3)

- Task distribution (`src/orchestrator/scheduler.ts`)
- Create `Task` from CLI; assign to N agents (competition optional)
- Track `TaskAssignment` lifecycle; deadlines and status updates

- Quality evaluation (`src/evaluation/checks.ts`, `src/evaluation/metrics.ts`)
- Implement checks: tests/lint/build/custom (shell commands)
- Metric registry; simple scoring and thresholding

- Merge strategies (`src/git/merge.ts`)
- Automatic: fast-forward or squash with conflict detection
- Manual: guidance + leave branches intact
- Cherry-pick: selective commit integration
- Rollback on failure with logs

- Competition basics (`src/orchestrator/competition.ts`)
- Run N agents in parallel on same task
- Evaluate and select winner by weighted metrics; summarize

Acceptance:

- End-to-end: spawn two agents on a task, run checks, pick winner, merge
- Auto-merge threshold and guardrails honored from config

### M4: Polish & Optimization (Week 4)

- Interactive setup wizard (`src/cli/setup.ts` using `inquirer`)
- Questions per spec; writes `crewchief.config.ts`
- Optional generation of example agent definitions

- Advanced competition features
- Custom metrics via user scripts
- Artifact diffing and reporting

- Performance
- Pane reuse; throttled capture
- Parallel worktree operations where safe

- Error recovery
- Transactional steps with rollback and cleanup
- Robust logging and crash reports

Acceptance:

- `crewchief setup` completes in < 2 minutes producing a valid config
- Competition reports are readable; failures recover without dangling panes/worktrees

## Risks and Mitigations

- Tmux availability differences on macOS/Linux: detect version; document requirements
- External CLI variability (Claude/Gemini): start with mock agents; add adapters behind `AgentType`
- Long-running processes and buffer growth: stream to file, rotate logs, apply backpressure
- Git conflicts and partial merges: transactional merge steps with rollback and clear guidance
