Worktree Management

Create isolated development environments from git branches

  • create <branch> - New worktree
  • list - Show all worktrees
  • use <worktree> - Switch context
  • merge <worktree> - Integrate changes
  • clean <worktree> - Full cleanup
Clean Flags:
--keep-branch - Preserve git branch
--keep-maproom - Keep search index
--keep-dir - Preserve directory
Cleanup Pipeline:
  • Remove directory
  • Remove git metadata
  • Delete branch (safe)
  • Cleanup maproom DB
  • Graceful error handling on partial failures

Semantic Code Search

AI-powered codebase indexing and intelligent retrieval

  • scan - Incremental index (default)
  • scan --force - Full reindex
  • search <query> - Semantic search
  • watch - Auto-index on changes
  • upsert - Update entries
  • generate-embeddings - Process vectors
Performance:
  • Parallel processing (4+ workers)
  • Configurable batch size tuning
  • Incremental indexing by default
  • Efficient database architecture

Agent Orchestration

Deploy and manage AI agents across environments

  • spawn <agents> <task>
  • agent list - Show active agents
  • agent message <id>
  • agent close <id>
Execution Modes:
  • iTerm2 - Visual terminal panes (macOS)
  • Headless - Background processes (any platform)
⚡ Note: Terminal backend configured via crewchief.config.js. iTerm2 mode requires macOS + iTerm2 installation.

📦 Binary Resolution Chain

Maproom binary discovery strategy

Resolution Order:

1. MAPROOM_BIN env var
↓
2. CREWCHIEF_MAPROOM_BIN env var
↓
3. Config file: crewchief.config.js
↓
4. Global install (npm/package manager)
↓
5. Packaged binary (fallback)
⚡ Best Practice: Set MAPROOM_BIN env var for explicit control in CI/CD pipelines.

⚙️ Configuration (crewchief.config.js)

repository: {
  mainBranch: "main",
  worktreeBasePath: "~/.crewchief/worktrees",
  maproomBinaryPath: "/usr/local/bin/maproom"
}

worktree: {
  copyIgnoredFiles: true,
  autoScanOnWorktreeUse: true
}

terminal: {
  backend: "iTerm2", // or "headless"
  iterm: { splitLayout: "vertical" }
}

evaluation: {
  autoMergeThreshold: 0.95
}

📍 Worktree Path Patterns

Flexible path resolution for worktree base locations

~ expansion: ~/my-worktrees → /Users/username/my-worktrees
<repo-name> placeholder: ~/.crewchief/<repo-name> → ~/.crewchief/my-repo
Absolute paths: /var/worktrees (direct use)
Relative paths: ./worktrees (relative to project root)
Default: ~/.crewchief/worktrees/<repo-name>
