@maproom/daemon-client

TypeScript client library for communicating with the maproom daemon via JSON-RPC 2.0. Eliminates process spawning overhead with 20-50x faster warm request performance.

2
First Request (Auto-Start)
On first search/scan request, daemon is automatically spawned and connected via stdio if not already running.
await client.search(...) → spawns daemon on first call
3
Active Communication
Client sends search/scan requests and receives results over persistent JSON-RPC connection.
Reuses connection for all subsequent operations
4
Health Checks & Monitoring
Periodic health checks validate daemon responsiveness and detect connection issues.
Automatic timeout handling after 30000ms (configurable)
5
Crash Detection & Auto-Restart
When daemon crashes, client detects failure and triggers restart with exponential backoff.
Circuit breaker prevents restart storms

Circuit Breaker State Machine

CLOSED
Normal operation. Requests proceed normally. Ready to auto-restart on failure.
OPEN
Too many restart attempts. Rejects new requests to prevent restart storms.
HALF_OPEN
Testing recovery. Allows single request to determine if daemon recovered.
State Transitions:
• CLOSED → OPEN: After 5 consecutive failures within timeout window
• OPEN → HALF_OPEN: After exponential backoff delay (5, 10, 20, 40s...)
• HALF_OPEN → CLOSED: Single request succeeds, reset failure counter
• HALF_OPEN → OPEN: Single request fails, exponential backoff continues

Connection Modes

Auto-Start (Default)
Client automatically spawns daemon on first request. Requires binaryPath to maproom executable. Simplest option for most use cases.
Connect to Existing
Client connects to an already-running daemon via stdio. Useful for long-lived daemon processes or shared instances.
Pool Mode
Multiple client instances share a single daemon. Reduces memory overhead for concurrent operations.
Custom Control
Manual lifecycle management with explicit start/stop/restart methods for advanced use cases.
